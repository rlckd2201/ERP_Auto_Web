# DECISIONS.md

## 확정 설계
- ERP 전표 작성은 운영서버가 아니라 172.17.30.243 자동 전표처리 PC Agent에서 수행한다.
- 업로드 이후 담당자가 브라우저를 닫아도 작업은 계속 진행되어야 한다.
- 엑셀의 여러 행은 개별 전표가 아니라 하나의 전표 안에 작성한다.
- 미지급금(원화)은 차변.
- 보통예금은 대변.
- 대변금액은 차변금액 총합.
- 회계일 기본값은 매월 20일.
- 출납처리여부는 체크 해제한다. 최초 준비 스냅샷에서 이름 있는 체크박스뿐 아니라 하단 폼 위치의 무명 `Check1`도 상태 확인에 사용한다. GDI가 체크박스 객체 자체를 제공하지 않거나 캐시가 끊어진 경우에는 이미 해제된 기본 상태를 임의 좌표 클릭으로 반전시키지 않고 그대로 진행한다.
- 담당자 계정 원천은 그룹웨어 MariaDB에서 읽되, 외부 DB에는 쓰지 않는다.
- 로그인 비밀번호와 세션은 신규 시스템 로컬 SQLite에 저장한다.
- 신규 계정 초기 비밀번호는 `wowjd12!@`이며 최초 로그인 시 변경을 강제한다.
- 비밀번호 찾기는 임시 비밀번호를 발급해 메일로 보내고 다음 로그인 시 변경을 강제한다.
- 완료 메일은 ERP 전표 저장 성공, 전표번호 확보, 출력 성공 이후에만 발송한다.
- Agent 기본 모드는 `dry-run`이며 실제 ERP 입력은 `-ErpMode real`로 명시한 경우에만 수행한다.
- 대승 수시결제 ERP 회계단위/전표관리단위는 1차로 `D1공장`을 사용한다.
- 기존 세금계산서 수신/Zoom/정기처리 기능과 신규 엑셀 업로드 전표처리는 분리한다.

## 적요 규칙
- 현재월 기준 지난달 수시결제 형식 사용.
- 예: 6월 작성 시 `5월 수시결제 - 업체명(업체코드)`.
- 미지급금(원화) 행 적요: `5월 수시결제 - 업체명(업체코드)`.
- 보통예금 행 적요: `5월 수시결제 - 신한은행`.

## ERP 관리항목 규칙
- 미지급금(원화) 행 관리항목: `거래처`.
- 미지급금(원화) 행 `거래처` 값은 엑셀 업체코드 우선, 없으면 업체명 사용.
- 첫 미지급금(원화) 행은 빈 관리항목값을 한 번 클릭한 뒤 `F9`를 누르고, 거래처번호 입력 → `Tab` 4회 → 아래 5회 → 위 2회 → `Tab` 3회 → `Enter` 2회 순서로 선택한다.
- 두 번째 미지급금(원화) 행부터는 빈 관리항목값 셀을 한 번 클릭하고 거래처번호를 입력한 뒤 Enter로 확정하고 다음 행으로 이동한다.
- 두 번째 행 이후 거래처값 Enter가 전표 행까지 이동시켰는지는 화면 경계에서 현재 행 Y 변화 또는 다음 행 선택 상태로 한 번 확인한다. 점 단위 UIA가 행번호를 제공하는 화면은 실제 이동 결과를 사용하고, GDI처럼 행번호를 제공하지 않는 화면은 이미 보낸 관리항목 Enter를 이동 신호로 사용해 마지막 완전 표시 좌표를 고정하며 추가 Down을 보내지 않는다.
- 한 화면에 보이는 마지막 완전 표시 행은 24/26 등 고정값으로 두지 않고 최초 캐시된 그리드 하단 경계와 행 간격으로 실행 중 계산한다. 행번호 지점 UIA는 가능한 화면에서 좌표를 보정하는 선택 기능이며 필수 조건이 아니다. 전체 행 수도 210으로 고정하지 않고 payload의 `rows_to_fill`을 사용한다.
- 화면 경계에서는 계산된 마지막 적요 좌표를 더블클릭한 뒤 행번호 지점을 다시 확인한다. UIA 행번호가 제공되면 값 확정 전후 현재 행 Y와 다음 행 선택 상태를 비교해 Enter/Down 방식을 고정한다. UIA 행번호가 제공되지 않으면 거래처값 Enter 뒤 `last_full_y`를 `enter` 방식으로 고정한다. 이후에는 전체 UIA 트리를 다시 열거하지 않고 같은 좌표를 마지막 행까지 재사용한다.
- 거래처 팝업의 검색조건은 `거래처번호`이며 결과 그리드 열 이름인 `거래처코드`를 검색조건이나 검색 버튼으로 취급하지 않는다.
- 위 첫 행/후속 행 전환은 미지급금(원화)에만 적용하며 보통예금 관리항목 입력 흐름은 기존 방식을 유지한다.
- 보통예금 행 관리항목: `계좌번호`, `금융기관지점`, `거래처`.
- 대승 보통예금 계좌번호: `140-000-948562`.
- 대승 보통예금 금융기관지점: `신한 수원금융센터`.
- 마지막 보통예금 행의 Ctrl+C 검증은 계정과목 또는 payload의 해당 행 적요가 정확히 복사되면 현재 행이 맞는 것으로 인정한다. 둘 다 불일치할 때만 계정과목 셀 중앙 `x=295`에서 한 번 더 확인한다.
- 운영 fast 경로(`ERP_SKIP_VISIBLE_ROW_SCAN=1`)에서는 위 GDI Ctrl+C 검증을 사용하지 않는다. 마지막 행 payload에 `계좌번호/금융기관지점`이 명시되면 기존 은행 좌표 입력으로 즉시 진행한다.
- 대승 보통예금 거래처: 빈 값.
- 대승정밀, 일강, 제이엠의 보통예금 계좌/지점은 담당자 확인 후 설정에 추가한다.
- 전표번호 자동 회수 전까지는 실제 저장/출력이 성공해도 완료 메일은 발송하지 않는다.

## 새 시스템 배치
- 신규 웹 시스템 폴더는 `excel_voucher_web/`이다.
- 서버는 FastAPI 기반으로 구현한다.
- 작업 상태와 Agent 큐는 SQLite에 저장한다.
- 업로드 원본은 `excel_voucher_web/data/uploads/`에 저장한다.
- 신규 웹 포트는 운영 기준 `8081`을 사용한다.
- 기존 전산팀용 `https://172.17.39.121:8080/`와 분리한다.
- 데이터 서버 전달 포트는 `18080`을 사용한다.

## 대승 수시결제 엑셀 형식
- 대상 샘플: `5월 대승 수시결제.xlsx`.
- 전표 작성 대상 행은 `수시결제현금` 시트에 있는 행만 사용한다.
- `수시결제어음`, `인터넷뱅킹`, `필수확인` 시트는 별도 전표 행을 만들지 않는다.
- `수시결제현금`은 코드/업체명/원본 적요/금액을 사용한다.
- 샘플 기준 `수시결제현금` H열의 저장값은 1~190 순번처럼 보이므로 전표 금액으로 사용하지 않는다.
- 이런 경우에만 같은 순번의 `인터넷뱅킹` D열 금액을 보조로 사용하되, 전표 행 기준은 계속 `수시결제현금`이다.
- ERP 붙여넣기 rows는 `계정과목\t\t차변\t대변\t적요` 형식을 사용한다.

## 관리항목 준비 감지와 243 배포 완료 기준
- 관리항목 자동입력은 payload 첫 행에 명시된 실제 관리항목명(`거래처`, `계좌번호`, `금융기관지점` 등)이 ERP 하단 관리항목 표에서 검출된 뒤에만 시작한다. 픽셀 변화 신호만으로는 준비 완료로 판정하지 않는다.
- Excel 범위 붙여넣기와 텍스트 클립보드 붙여넣기는 동일한 반복 감지 함수를 사용하며, 준비 완료 뒤 항상 1행부터 관리항목 입력을 시작한다.
- 마지막 보통예금 fast 좌표는 실제 관리항목명이 검출된 스냅샷에서만 계산한다.
- `243 최신 적용`은 명령 큐 등록이 아니라 Agent의 실제 완료와 설치 대상 담당자 프로그램의 SHA-256 일치가 확인된 시점을 완료로 표시한다.
