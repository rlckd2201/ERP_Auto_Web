# DECISIONS.md

## 확정 설계
- ERP 전표 작성은 운영서버가 아니라 172.17.30.243 Agent에서 수행한다.
- 업로드 이후 담당자가 브라우저를 닫아도 작업은 계속 진행되어야 한다.
- 엑셀의 여러 행은 개별 전표가 아니라 하나의 전표 안에 작성한다.
- 미지급금(원화)은 차변.
- 보통예금은 대변.
- 대변금액은 차변금액 총합.
- 회계일 기본값은 매월 20일.
- 담당자 계정 원천은 그룹웨어 MariaDB에서 읽되, 외부 DB에는 쓰지 않는다.
- 로그인 비밀번호와 세션은 신규 시스템 로컬 SQLite에 저장한다.
- 신규 계정 초기 비밀번호는 `wowjd12!@`이며 최초 로그인 시 변경을 강제한다.
- 비밀번호 찾기는 임시 비밀번호를 발급해 메일로 보내고 다음 로그인 시 변경을 강제한다.
- 완료 메일은 Agent가 출력 제출 성공을 보고한 뒤 발송한다.
- 기존 세금계산서 수신/Zoom/정기처리 기능과 신규 엑셀 업로드 전표처리는 분리한다.

## 적요 규칙
- 현재월 기준 지난달 수시결제 형식 사용.
- 예: 6월 작성 시 `5월 수시결제 - 업체명(업체코드)`.
- 미지급금(원화) 행 적요: `5월 수시결제 - 업체명(업체코드)`.
- 보통예금 행 적요: `5월 수시결제 - 신한은행`.

## 새 시스템 배치
- 신규 웹 시스템 폴더는 `excel_voucher_web/`이다.
- 서버는 FastAPI 기반으로 구현한다.
- 작업 상태와 Agent 큐는 SQLite에 저장한다.
- 업로드 원본은 `excel_voucher_web/data/uploads/`에 저장한다.
- 신규 웹 포트는 운영 기준 `8081`을 사용한다.
- 기존 전산팀용 `https://172.17.39.121:8080/`와 분리한다.
- 데이터 서버 전달 포트는 `18080`을 사용한다.

## 대승 수시결제 엑셀 형식
- 대상 샘플: `5월 대승 수시결제.xlsx`.
- 전표 작성 대상 행은 `수시결제현금` 시트에 있는 행만 사용한다.
- `수시결제어음`, `인터넷뱅킹`, `필수확인` 시트는 별도 전표 행을 만들지 않는다.
- `수시결제현금`은 코드/업체명/원본 적요/금액을 사용한다.
- 샘플 기준 `수시결제현금` H열의 저장값은 1~190 순번처럼 보이므로 전표 금액으로 사용하지 않는다.
- 이런 경우에만 같은 순번의 `인터넷뱅킹` D열 금액을 보조로 사용하되, 전표 행 기준은 계속 `수시결제현금`이다.
- ERP 붙여넣기 rows는 `계정과목\t\t차변\t대변\t적요` 형식을 사용한다.
