# SESSION.md

## 운영 원칙
- 컨텍스트 절약 최우선.
- 긴 분석, 로그, 결정사항, 진행상황은 채팅에 누적하지 말고 MD 파일에 갱신한다.
- 작업 시작 전 관련 MD와 graphify 결과를 먼저 읽는다.
- 필요한 파일만 좁게 열어 진행한다.
- 코드 구조, 의존관계, 영향범위 확인은 graphify를 적극 활용한다.
- 답변은 핵심만 짧게 한다.
- 자세한 인수인계 정보는 MD에 남긴다.
- 작동하던 기존 기능은 임의 삭제 금지.
- 삭제가 필요하면 먼저 질문한다.

## 신규 목표
재정부서 회계업무 자동화 확대 전개.
세금계산서 메일 수신 기반이 아니라, 담당자가 엑셀을 업로드하면 전표 처리를 수행하는 기능을 개발한다.

## 기본 흐름
1. 담당자가 엑셀 업로드.
2. 업로드는 드래그 앤 드롭 또는 파일 선택으로 가능.
3. 업로드 시 회계일 입력/수정 가능.
4. 서버는 작업을 큐에 등록.
5. 담당자는 창을 닫아도 됨.
6. 172.17.30.243 담당자 PC Agent가 자동으로 ERP 전표 작성/출력 수행.
7. 출력 제출 성공 후 담당자에게 완료 메일 발송.

## 계정/메일 방향
- 그룹웨어 MariaDB `ksystem_yundong`의 `gw_emp`, `ds_t_emp`에서 담당자 계정/메일/부서 정보를 읽는다.
- 외부 DB는 읽기 전용으로만 사용한다.
- 로컬 시스템 초기 비밀번호는 `wowjd12!@`.
- 최초 로그인 및 임시 비밀번호 로그인 시 비밀번호 변경을 강제한다.

## 이번 세션 구현 범위
- 기존 `web_v1` 기능은 수정하지 않는다.
- 새 시스템은 `excel_voucher_web/`에 독립 구성한다.
- 1차 구현은 엑셀 업로드, 전표 payload 생성, 서버 작업 큐, Agent 폴링 API, 담당자용 웹 화면까지 포함한다.
- 실제 ERP 화면 자동입력은 `app/agent_adapter.py`와 `agent/agent_worker.py`에 분리된 후속 구현 지점으로 둔다.
